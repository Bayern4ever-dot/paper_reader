## 摘要
我们提出了一种基于以原点为中心的图（OCG）生成任务断点的方法，旨在将面向目标的活动录制划分为任务单元，以便在虚拟现实（VR）环境中进行自适应回放。随着增强现实（AR）/虚拟现实（VR）头戴式显示器（HMD）的发展，关于自适应教程和创作工具的研究日益活跃，但现有的任务分割方法主要依赖于手动标注，或局限于 2D 视频，这限制了它们在 3D VR 场景中的适用性。在我们的方法中，使用结构化时空场景图（STSG）记录具有明确任务边界的装配场景，并利用 OCG 追踪中心对象的变化和新组的形成，从而自动生成任务断点。一项用户研究收集了用户感知的任务断点以建立基准真相（GT），与算法检测到的断点进行对比显示出高度一致性，并证实了其在支持自适应回放方面的准确性。所提出的任务分割方法为根据用户熟练度和进度动态调整 VR 回放奠定了基础，并具有扩展到各种 VR 录制的自动时间线分割系统的潜力。

## Abstract
We propose a method for generating task breakpoints based on an Origin-Centric Graph (OCG) to segment goal-oriented activity recordings into task units for adaptive playback in Virtual Reality (VR) environments. With the development of Augmented Reality (AR)/VR head-mounted displays (HMDs), research on adaptive tutorials and authoring tools has become active, but existing task segmentation methods mainly rely on manual annotation or are restricted to 2D video which limits their applicability to 3D VR contexts. In our approach, assembly scenarios with clearly defined task boundaries are recorded using a structured spatio-temporal scene graph (STSG), and the OCG is employed to track changes in the central object and the formation of new groups, thereby generating task breakpoints automatically. A user study collected user-perceived task breakpoints to establish ground truth (GT), and comparison with the algorithm-detected breakpoints demonstrated high agreement and confirmed accuracy in supporting adaptive playback. The proposed task segmentation method provides a foundation for dynamically adjusting VR playback according to user proficiency and progress, with potential for extension into automatic timeline segmentation systems for diverse VR recordings.

*   **研究动机**：随着空间计算（VR/AR）的发展，3D 空间视频录制成为学习复杂任务（如装配）的重要工具。然而，现有的任务分割方法要么依赖耗时的人工标注，要么局限于 2D 视频分析，难以捕捉 3D 环境中复杂的物体间关系和用户交互。
*   **核心问题**：如何在 VR 环境中自动、分层地（粗粒度与细粒度）分割目标导向的任务录制，以支持根据用户熟练度调整进度的“自适应回放”（Adaptive Playback）。

### 2. 方法论
该方法的核心在于将动态的交互数据转化为结构化的图表示，并利用拓扑变化识别任务边界。
*   **时空场景图 (STSG)**：在录制阶段，系统以 60fps 记录用户手部（21个关节的 6DoF 信息）和物体（位置、旋转、类别）的状态。通过“手部邻接矩阵”记录抓取关系，通过“物体邻接矩阵”记录物体间的物理连接或工具操作关系。
*   **以原点为中心的图 (OCG)**：
    *   **原点选择**：基于最终装配状态，利用**度中心性 (Degree Centrality)** 算法识别出连接数最多的核心部件作为“原点对象”。
    *   **权重计算**：利用广度优先搜索 (BFS) 计算各部件到原点的最短路径距离，以此量化部件在结构中的重要性。
*   **任务断点生成算法**：
    *   **细粒度断点 (Fine)**：基于三种触发规则：1. 直接连接到原点；2. 中心对象更新（当前装配组中权重最高的部件发生变化）；3. 新子组形成。
    *   **粗粒度断点 (Coarse)**：将共享相同中心对象或相同功能目标的连续细粒度单元进行合并。
    *   **行为精炼 (Refinement)**：根据手部释放物体的时刻（Hand Adjacency Matrix 回归零）微调断点时间，使其符合人类对“动作完成”的感知。

### 3. 实验设计
*   **实验场景**：选择了两种不同复杂度的装配任务：
    1.  **自行车装配**（简单）：部件少、结构中心化程度高、并行步骤多。
    2.  **无人机装配**（复杂）：部件多、存在大量重复性子任务（如4个螺旋桨）、结构相对分散。
*   **基准真相 (Ground Truth, GT)**：招募 24 名用户观看录像并手动标注他们感知的细粒度和粗粒度断点。使用 **DBSCAN 聚类算法**处理用户标注的偏差，取聚类中值作为 GT。

### 5. 实验数量与充分性
*   **实验规模**：
    *   **初步研究**：12 人，用于确定用户分割任务的心理模型和准则。
    *   **正式评估**：24 人，生成了大量的标注数据用于验证算法。
*   **充分性评价**：实验设计较为充分。通过对比简单与复杂两种任务，验证了算法的鲁棒性；引入 DBSCAN 处理人类感知的模糊性，保证了 GT 的客观性。但实验目前仅限于“装配类”任务，对于非结构化或高度动态的任务（如烹饪、手术）尚未验证。

### 6. 主要结论与发现
*   **高准确性**：在细粒度断点检测上，F1 分数高达 **0.98**；在粗粒度断点上为 **0.90**。
*   **时间误差小**：自行车任务的 MAE 低至 0.44s，无人机任务（较复杂）的 MAE 为 1.38s，均在人类感知的可接受范围内。
*   **认知一致性**：OCG 能够有效捕捉用户在执行任务时的心理分层，粗粒度断点成功合并了重复的子动作（如连续安装四个螺丝），这对于自适应教程非常有用。

### 7. 优点
*   **自动化程度高**：无需人工标注，只要 VR 内容提供基础的物体交互信息即可自动生成时间轴。
*   **分层结构**：同时提供粗、细两种粒度，能够满足不同熟练度用户的回放需求（新手看细步骤，熟练者看大环节）。
*   **符合人类直觉**：通过 OCG 捕捉“结构中心”和“动作完成时刻”的精炼步骤，使分割结果非常接近人类的自然感知。

### 8. 不足与局限
*   **任务类型受限**：目前仅适用于有明确最终结构状态的“装配式”任务。对于没有固定终态的任务，难以定义“原点对象”。
*   **离线依赖**：OCG 的生成依赖于任务完成后的最终状态，这意味着该方法目前主要用于“录制后的自动处理”，而非完全实时的在线分割。
*   **AR 扩展性挑战**：在真实世界（AR）中，获取精确的物体间连接关系和手部 6DoF 数据比在纯虚拟环境（VR）中困难得多，需要更强的计算机视觉算法支持。
